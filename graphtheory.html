Introduction to Computer Networks

Networks LAN WAN TCP/IP

What is a Computer Network?

A computer network is a group of interconnected devices that can share data, resources, and services. These devices, known as nodes, include computers, printers, phones, servers, and more. Networks help us send emails, access websites, share files, stream videos, and communicate globally.

Key Network Characteristics

Computer networks have several important characteristics that define how they operate:

  • Connectivity: Allows devices to communicate with each other
  • Resource Sharing: Enables sharing of files, printers, and other resources
  • Communication: Facilitates the exchange of information
  • Scalability: Can be expanded to include more devices
  • Reliability: Provides redundancy to ensure consistent operation

Importance of Computer Networks

Computer networks are fundamental to modern technology and society in many ways:

  • Information Sharing: Networks allow for quick and efficient exchange of information
  • Resource Utilization: They enable optimal use of hardware and software resources
  • Reliability: Networks can provide redundancy to ensure data availability
  • Cost Efficiency: Sharing resources over a network can reduce hardware and software costs
  • Communication: They enable various forms of communication like email, messaging, and video calls

Types of Networks

Computer networks come in various sizes and configurations, each designed for specific purposes and environments. Here are the main types:

Network Classifications by Size and Scope

Type Description Example
LAN (Local Area Network) A network within a small area like a home, school, or office Your school's computer lab
WAN (Wide Area Network) Covers a large area, often made up of multiple LANs The internet
WLAN (Wireless LAN) A LAN that uses wireless connections Wi-Fi at a coffee shop
PAN (Personal Area Network) Very small, used for one person's devices Bluetooth between phone and smartwatch
MAN (Metropolitan Area Network) Covers a city or campus area City-wide Wi-Fi or cable network

Network Topologies

Topology refers to how devices are arranged in a network. Some common types:

  • Bus Topology: All devices connected to one main cable
  • Star Topology: All devices connect to a central hub or switch
  • Ring Topology: Each device connects to two others in a loop
  • Mesh Topology: Devices are interconnected with many paths between them

Star topology is most common in modern networks due to reliability and easy troubleshooting.

Network Devices and Connections

Various hardware components work together to build a functioning network. Understanding these devices is essential for network design and troubleshooting.

Key Network Devices

Device Purpose
Router Connects different networks (e.g., home network to the internet)
Switch Connects multiple devices in a LAN and manages traffic
Modem Converts signals from your ISP into data your router can use
Access Point Adds Wi-Fi capabilities to a wired network
Firewall Protects the network by blocking unauthorized access

Types of Connections

  • Wired Networks: Use cables like Ethernet for faster and more stable connections
  • Wireless Networks: Use radio waves (like Wi-Fi and Bluetooth) for mobility and convenience

Network Models

Model Description Example
Client-Server Clients request data from a central server Websites, school database
Peer-to-Peer (P2P) Devices share data directly Bluetooth file sharing, torrents

Network Addressing and Protocols

For devices to communicate on a network, they need addressing systems and protocols that govern how data is transmitted.

Network Addressing

  • IP Address: A unique number that identifies a device on a network (like a digital home address)
  • MAC Address: A permanent hardware address unique to each device's network interface

Network Performance Concepts

  • Bandwidth: The amount of data that can be transmitted in a set amount of time (like the width of a road)
  • Latency: The delay before data starts moving (like the reaction time of a car)

Common Internet Protocols

Protocol Function
HTTP/HTTPS Transfers web page data
FTP Uploads and downloads files
TCP/IP Core set of rules for communication on the internet
DNS Translates website names into IP addresses

Real-World Network Examples

Let's look at how networks function in everyday scenarios:

Sending an Email

Your device connects to a mail server using protocols like SMTP and IMAP. The email travels across multiple networks before reaching its destination.

Streaming a Video

Your device uses TCP/IP to request and receive video packets from a server. The data is broken into packets, transmitted across the internet, and reassembled on your device.

Browsing a Website

Your browser uses HTTP to load the content and DNS to find the site's IP. The web server responds by sending the requested web page data back to your device.

Think About It

Consider the networks you use every day:

  • How many different networks do you connect to in a typical day?
  • What types of data do you transmit and receive?
  • How would your daily activities be affected if these networks were unavailable?
